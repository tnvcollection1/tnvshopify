**Code Architecture**


**Key Technical Concepts**
-   **Headless E-commerce**: The storefront acts as a head for content managed in Shopify. Data is synced to a middle-layer database (MongoDB) and served via a FastAPI backend.
-   **Shopify GraphQL API**: Used for fetching nested data like navigation menus, which is not easily accessible via the REST API.
-   **Multi-tenancy by Hostname/Query**: A single frontend codebase serves multiple stores by detecting the domain or a URL parameter, then loading the correct configuration and fetching store-specific data.
-   **Vercel Deployment**: The standard workflow for deploying the standalone Vite/React frontend, including managing custom domains and environment variables.

-   **shopify_products**: Stores synced products from Shopify.

-   **shopify_menus**: (New) Stores synced navigation menus from Shopify.
